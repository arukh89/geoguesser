Alert Investigation Geo Detective Mini‑App: Map hidden, Matrix theme inconsistencies

SEV-2 Started: 2025-12-12 14:43:04 UTC
On-call: Asep Asep

Executive Summary

Because Leaflet CSS was not guaranteed to load client-side and the overlay container masked the map, the map UI rendered but appeared blank; simultaneously, several components (e.g., Leaderboard, some buttons) bypassed Matrix tokens, producing an off-theme UI, and Matrix rain respected system reduced‑motion defaults with no in‑app override, so it often appeared off. Setting the Mapillary public token was also required to fetch coverage/imagery.

Root Cause Analysis

Suspected root cause:

  • Map blank: Leaflet CSS imported via a conditional require in a helper, not a guaranteed top-level CSS import within client components → tiles and layers didn't style/paint correctly under Next 15 in some builds.
  • Theme drift: Leaderboard and some UI used non‑Matrix palettes (white/purple) and ignored Matrix tokens/utilities.
  • Matrix rain off: MatrixRain obeys prefers-reduced-motion and lacked an explicit in‑app toggle.
  • Imagery: Mapillary endpoints require NEXT_PUBLIC_MAPILLARY_TOKEN; missing env led to fallback/non‑interactive imagery.

Evidence

Code Reference

Leaflet setup found in src/lib/leaflet.config.ts and imports in WorldMap.tsx, ResultsScreen.tsx. Mapillary viewer uses process.env.NEXT_PUBLIC_MAPILLARY_TOKEN in MapillaryViewer.tsx; API coverage tiles use the same in src/app/api/imagery/mapillary/route.ts.

WorldMap.tsx → import '@/lib/leaflet.config' (no direct 'leaflet/dist/leaflet.css' import)
MapillaryViewer.tsx → Viewer({ accessToken: process.env.NEXT_PUBLIC_MAPILLARY_TOKEN! })

Proposed Fix

  1. Immediate Mitigation (Map): Add explicit CSS import in client components.
    Steps: In src/components/game/WorldMap.tsx and ResultsScreen.tsx, add import "leaflet/dist/leaflet.css"; at top. Keep height:100% on containers.
  2. Short‑term (Matrix rain + theme): Add a settings toggle to override reduced‑motion and ensure Matrix tokens/ utilities wrap all UI (Buttons, Cards, Leaderboard).
    Steps: Expose useMatrix().setSettings({ rain: true }) via a small menu in the header; refactor Leaderboard to use Matrix classes (mx-panel, mx-border, --accent).
  3. Env and Imagery: Set Mapillary envs and use server‑only token for API route.
    Steps: Add NEXT_PUBLIC_MAPILLARY_TOKEN in .env.local; switch API route to MAPILLARY_SERVER_TOKEN server‑only.
  4. Long‑term (Global Matrix System): Finalize a Matrix design system (fonts VT323 / Share Tech Mono, tokens, components, animations) and apply across app.
    Validation: Visual QA across screens (home, play, results, final) on small portrait viewports per Mini App guidelines.
