Executive Summary

Because SpacetimeDB and Next.js both default to port 3000, running both locally caused a port collision. Separately, home/results layout issues came from missing full-viewport background layering and oversized image overlays. We configured SpacetimeDB on 127.0.0.1:3010 and pointed the app via .env.local, rendered MatrixRain as a fixed, full-viewport background globally, centered startup/home content, added a Mapillary-style left sidebar, and constrained the mission image preview to ~1/6 viewport height so the map remains fully visible. All changes were done locally with hot reload; no GitHub changes.

Root Cause Analysis

Suspected root cause:

  • Primary Hypothesis: Default port collision (Next dev on 3000 vs SpacetimeDB default 3000) and missing CSS constraints for background/overlay sizing.
  • Contributing Factors: App fallback STDB URI defaulted to 3000; Matrix rain previously not guaranteed globally; image overlay lacked a hard height cap.
